## Figure 1 — The expectancy feedback loop: open vs. closed  ·  Critical
**Heuristic:** MC
**Concept (one sentence):** A teacher prediction allocates task rigor and wait time, the student reads the allocation and adjusts effort, the outcome confirms or challenges the prediction, and looping back — a label converts the open loop (updated by fresh evidence) into a closed one (reinforced regardless of evidence).
**Figure type:** Cyclical diagram with two labeled versions (open and closed).

**Block 1 — Illustrae paste block:** Create a cyclical diagram on a white single-column canvas, 89mm wide, 300 DPI, flat vector. Show a closed ring of four stages connected by single-headed arrows returning to the start: teacher forms a prediction; task rigor and wait time allocated accordingly; student reads the allocation and adjusts effort; outcome confirms or challenges the prediction; arrow back to the prediction stage. Present two versions of this same ring. Version one (open loop): the return path carries fresh evidence that updates the prediction — render this return path in bluish green as the healthy version. Version two (closed loop): the return path reinforces the prediction regardless of evidence — render this path in vermillion as the failure version, with a label element marking that a fixed label is what closes the loop. The payload is that the same loop runs two ways and a label is the switch. Use blue for the prediction node (the dominant concept), sky blue for the intermediate stages, bluish green for the open/updating return path, vermillion for the closed/reinforcing return path, light gray for stage arrows. Maximum eight labeled components. One-pixel strokes, white background, request an unannotated vector with no baked-in text.
